What is HTML?

  • HTML stands for Hyper Text Markup Language
  • HTML describes the structure of a Web page
  • HTML consists of a series of elements

What is an "Element"?

  • HTML elements tell the browser how to display the content
  • HTML elements are represented by tags
  • Browsers do not display the HTML tags, but use them to render the content of the page
  • HTML tags contain content labeled by a name such as "heading", "paragraph", "table", and so on

What is a "tag"?

HTML tags usually come in pairs. The opening tag (<>) and the closing tag (</>) . Some tags don't require a closing tag. Those are self-closed tags.
HTML tags are element names surrounded by angle brackets:

<tagname> content goes here...</tagname>

Structure

So, now that we understand each other and without any further delay, let's dive into making our first webpage! 😃
When you create a new file, either in notepad or in Visual Studio Code, you need to save it as a html file. If the file's extension is not (.html), the browser won't be able to read the html code.

The first line of code must be the <!DOCTYPE html> declaration.
This represents the document type, which in our case is html and it helps browsers to display our web pages correctly.
After this, we are ready to start our html document.

The HTML document itself begins with <html> and ends with </html>.
In between we can add the <head>...</head> element. Inside the head element, we can add an important tag, the <title>. ( remember to add the closing tag after you write the title 😉)
The title shows the title of your webpage on your browser's tab.

Elements

Headings

Heading is a title. It may be the title of your article for example.
Headings come is 6 sizes and they are defined by tags from <h1> for the most important title to <h6> for the least important.

For example:
This site's title is typed as:<h1>Build your own website</h1>
The chapters are typed as:<h3>Headings</h3>

Paragraphs

Every website has text. You can type your text using the p element. The browser automatically adds some margin and font to your text but you can change that later.
Note, that your text editor will ignore any extra spaces or lines you add. If you want to enter a new line to your text use the br tag.

For example:
<p>This is a paragraph which <br>
continues to a second line </p>

Text Formatting

HTML contains several elements for defining text with a special meaning:

  • <b> - Bold text
  • <em> - Italic text
  • <mark> - Marked text
  • <small> - Small text
  • <del> - Deleted text
  • <ins> - Underlined text
  • <sub> - Subscript text
  • <sup> - Superscript text
  • For Example:
  • <b>Hello</b>: Hello
  • <em>Hello</em>: Hello
  • <mark>Hello</mark>: Hello
  • <small>Hello</small>: Hello
  • <del>Hello</del>: Hello
  • <ins>Hello</ins>: Hello
  • <sub>Hello</sub>: Hello world
  • <sup>Hello</sup>: Hello world

HTML Quotation

Sometimes you need to quote something, use a abbreviation or an acronym, like "HTML", "CSS", "Mr.", "Dr.", "etc", "ATM".
In these cases, Html provides tags that you can use, such as "q" and "abbr".

The <q>tag defines a short quotation. Browsers normally insert quotation marks around the quotation.

The <abbr>tag defines abbreviation or an acronym. Marking abbreviations can give useful information to browsers, translation systems and search-engines.
You can use the global title attribute to show the description for the abbreviation/acronym when you mouse over the element.

For Example:
Learning never exhausts the mind(<q>Learning never exhausts the mind</q> )
HTML is easy to learn. (<abbr title="Hyper Text Markup Language">HTML</abbr>is easy to learn. )

Links

Every now and then you will want to add some links, either external, leading to some other website or internal, leading to some other content inside your webpage.
You add link with the a tag. The link element needs at least one parameter, which you type inside the opening tag and that is the url of your link. The link's address can be on the internet or local, on your computer.

-10-

If you don't add this parameter your link simply won't work. Between the opening and closing tag you write a text that you want to appear.

There are many parameters that for every html element and they are called attributes.
However the attribute that we need right now for our links to work is href="url-address".

For example:
<a href="http://mycodingjourney...."> This is a link to my website </a>

For a local address:

  • if the destination file is in the same folder as your current file, you type: href="./the link's location"
  • if the destination file is one folder above your current file, you type: href="../the link's location"
  • Otherwise, you type the whole local address of your file's destination on your computer.

Media

Images

How much boring would be a website without pictures, right?
Fear not, because we are able to add any kind of image on our site!😃
We add pictures with the tag img. This tag is self-closed, it doesn't require a closing tag.

It needs at least one attribute, the src="", which works the same way as the href for the links. Another attribute that we can use is the alt="". The alt attribute is useful because we can add a small text,invisible to the viewers, describing the picture, in case the browser doesn't load the picture. Also screen readers will read out loud the picture's description. You can also specify the width="" and the height="" of your picture either in pixels or on percentage, to make sure that the image has the dimensions that you like.
Let's see an example:

Audio

To play an audio file in HTML, use the <audio> element. Browsers support .wav, .ogg, .mpeg, .mp3 and .mp4 audio formats.


The audio tag accepts several attributes that can come handy:

  • src="": Well, that's required and it specifies the URL of the audio file
  • controls: Required and it specifies that audio controls should be displayed (such as a play/pause button)
  • autoplay: Specifies that the audio will start playing as soon as it is ready
  • loop: Specifies that the audio will start over again, every time it is finished
  • muted: Specifies that the audio should be muted

For Example:

<audio controls src="url link"></audio>

-14-

Video

The HTML video element is used to show a video on a web page and it has same attributes as the audio tag.
You can also add width and height attributes, if you want to specify the video's size.

For Example:

<video controls width="320px" src="url link"></video>

Tables

Sometimes you might need to display info like prices, scores, data etc. These kind of data are tabular so the best way to present them is through a table, like an excel table.
Before we display the data we must manually create the "table" element along with the rows and columns.

So first we create the table by table tag and then nested inside the table we create the rows with the tr tag.
Nested in the rows, we create the columns, which contain our data, by using the td tag.
If our data expand to next columns, we use the colspan="" attribute. The number tells in how many columns the data will span and it has to be bigger than 1.

For Example:

Forms

An HTML form is used to collect user input. Every time you answer to a questionnaire or you order food from your favorite restaurant, you fill in a form.
The form element defines a form that is used to collect user input. An HTML form contains form elements. Form elements are different types of input elements, like:
text fields, checkboxes, radio buttons, submit buttons, and so on.

The form needs two attributes: action="" contains the target-location where the form will be submitted. method="POST or GET" tells which
method the browser will use to send the information collected by the form. POST method is more secured and it's used for sending sensitive information.

-17-

Inputs

Then we can add our inputs, nested in the form.
The self-closed input tag is the most important form element, because it displays several inputs,depending on the type attribute:

  • <input type="text">: It creates a text field.
  • <input type="email">: It creates a text field that accepts a valid e-mail address.
  • <input type="password">: It creates a text field that will replace the text with dots.
  • <input type="number">: It creates a little box where the user can add numbers. We can add the step="1" attribute to add clickable arrows that change the numbers
  • <input type="submit">: It creates a button for sending the form. We can add the value="" attribute to write a text which will be shown
    otherwise the default text is submit.

The input tag needs two attributes that are sent as pairs to the server when the form is submitted. The name="" contains the name of the input field. The value="" contains the data.
If we leave it empty, the input's value will be the user's data. But if we write something, our text will be sent as input instead.

-18-

Textarea

The input type="text" will add a small text field. If we want the user to write a bigger text, then we need the textarea tag.
We can define its size by adding cols="" and rows="".

Label

Another important tag is the label. That shows the text, connected with our input. To connect the input with label we need an id attribute on input and a for attribute on label.
An advantage for using a label element is that when the user clicks the text, the cursor will appear inside the input area.

-19-

Form Validation

We always need to validate the user's input so we catch errors before the user submits the data to our server.
The Html5 language has build-in form validation that saves time and make forms user-friendly.

  • The required attribute throws an error if the input is empty.
  • If we set a min="" and/or max="" number, there will be an error if the user tries to add something else.
  • For the text areas, if we don't want huge messages, we can set a cap-limit by adding minlength="" and/or maxlength="" attributes.
  • The email attribute will automatically check if the address contains a @ symbol.

Semantic HTML

On the structure chapter we saw how the elements are contained into body tag which is contained into html tag.
Think these as "boxes" which contain the information. img tag for instance is a semantic element that contains a link to your image.
Semantic elements provide content for the information and it's good to use them because:

  • gives structure to your code
  • make your code easily understandable by other developers
  • search engines understand better the content
  • makes the website more accessible to people who use screen-readers

-22-

The header element usually contains the navigation links, intro content or the site's heading.

The nav element contains your navigation links, menus, links etc.

The main element contains your main content. A nav element cannot contain a main element but the main can contain a nav.

The footer element is separated from the main element and it's located usually at the end of the webpage, containing info as contact, address, sitemap etc.

As we said, it's good to use these "boxes" as they group up your information and you can then easily style them. There are semantic elements that can be nested inside the main element.

-23-

The section element is a general element that groups up content with similar subject. A website's mainpage could be split into different sections for news, items, contact etc.

The article element contains information as your text, images, lists etc.

The aside element can be used alongside article and section for providing extra info.

The figure element can contain any form of media.

The figcaption element is nested inside the figure and it contain text describing the media in the figure element.
